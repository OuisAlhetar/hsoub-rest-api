# مشروع REST API باستخدام Laravel

هذا المشروع هو تطبيق عملي تم تطويره كجزء من دورة "تطوير تطبيقات PHP المتقدمة" المقدمة من أكاديمية حسوب. يهدف المشروع إلى بناء واجهة برمجية (REST API) لإدارة الدروس، المستخدمين، الوسوم، والعلاقات بينهم باستخدام إطار العمل Laravel.

## حول المشروع

يحتوي هذا المشروع على واجهات برمجية تتيح لك تنفيذ العمليات الأساسية (CRUD) على الكيانات التالية:
- المستخدمون (Users)
- الدروس (Lessons)
- الوسوم (Tags)
- ربط الدروس بالوسوم والمستخدمين

كما يتضمن المشروع نظام مصادقة باستخدام Laravel Sanctum، وسياسات صلاحيات (Policies) للتحكم في العمليات المسموح بها لكل مستخدم.

## المتطلبات

- PHP 8.1 أو أحدث
- Composer
- قاعدة بيانات SQLite (أو MySQL مع ضبط الإعدادات)
- Node.js و npm (لتجميع الموارد الأمامية إذا لزم الأمر)

## خطوات التشغيل

1. **استنساخ المشروع:**
   ```bash
   git clone <رابط المستودع>
   cd hsoub-rest-api
   ```

2. **تثبيت الاعتمادات:**
   ```bash
   composer install
   npm install
   ```

3. **إعداد ملف البيئة:**
   - انسخ ملف البيئة الافتراضي:
     ```bash
     cp .env.example .env
     ```
   - عدّل إعدادات قاعدة البيانات إذا لزم الأمر.

4. **توليد مفتاح التطبيق:**
   ```bash
   php artisan key:generate
   ```

5. **تشغيل الترحيلات وزرع البيانات:**
   ```bash
   php artisan migrate --seed
   ```

6. **تشغيل الخادم:**
   ```bash
   php artisan serve
   ```

## شرح الواجهات البرمجية

- جميع الواجهات متوفرة عبر المسار `/api/v1/`
- أمثلة على بعض المسارات:
  - `POST /api/v1/login` : تسجيل الدخول والحصول على توكن
  - `GET /api/v1/lesson` : جلب جميع الدروس
  - `POST /api/v1/lesson` : إضافة درس جديد (يتطلب صلاحيات)
  - `GET /api/v1/user/{id}/lessons` : جلب دروس مستخدم معين
  - `GET /api/v1/tag/{id}/lessons` : جلب الدروس المرتبطة بوسم معين

## المصادقة والصلاحيات

- يستخدم المشروع نظام المصادقة عبر التوكن (Sanctum).
- بعض العمليات تتطلب أن يكون المستخدم مسجلاً كـ "مدير" (admin).
- تم تطبيق سياسات (Policies) للتحكم في من يمكنه إنشاء أو تعديل أو حذف المستخدمين والدروس.

## المجلدات المهمة

- Models : نماذج البيانات (User, Lesson, Tag)
- Controllers : المتحكمات الخاصة بالواجهات البرمجية
- Policies : سياسات الصلاحيات
- migrations : ملفات الترحيل
- seeders : ملفات زرع البيانات التجريبية
